1. Data Minimization (OmniPOS)

The frontend captures the transaction intent but immediately passes it to the secure backend. In a production environment, CC numbers are swapped for secure network tokens before they even hit the switchboard, keeping the frontend completely out of PCI scope.

2. Bifurcation (POSitiveRails)

This is the "Brain". Google Cloud analyzes the transaction flag (e.g., B2B, Crypto, Retail) and routes the API call down the single most profitable and efficient pipe. It acts as an absolute firewall between the public internet and the payment processors.

3. Non-Custodial Processing

OmniPOS never touches the actual fiat currency or stablecoins. The processors (PayRoc, Priority, BitPay) move the money directly to the merchant's bank account. OmniPOS simply orchestrates the data and collects a software/gateway fee.

4. The Immutable Ledger

Firestore acts as the paper trail. Toxic data (full CC numbers, PII) is stripped away. Only the exact server timestamp, the routing rail used, the transaction ID, and the status are permanently written to the database for auditing and dashboard analytics.
